## 6. Знакомство с СУБД MariaDB и движками хранения

**Освоенные навыки:**  
Изучена СУБД MariaDB как форк MySQL, рассмотрены основные движки хранения (Storage Engines), их особенности и области применения.

**Основные сведения:**

- **MariaDB** — реляционная СУБД, созданная как продолжение MySQL после приобретения MySQL компанией Oracle. Сохраняет совместимость с MySQL API и протоколами, содержит расширенный набор движков хранения и улучшенную производительность.

**Рассмотренные движки хранения (Storage Engines):**

| Движок | Особенности | Применение |
|--------|-------------|------------|
| **InnoDB** | Стандартный движок MySQL/MariaDB. Поддерживает транзакции (ACID), внешние ключи (FOREIGN KEY), блокировки на уровне строк, MVCC (многоверсионность). Наиболее универсальный движок для OLTP-систем. | Системы с высокими требованиями к целостности данных, транзакционные приложения, интернет-магазины, банковские системы. |
| **MyISAM** | Более старый движок, не поддерживает транзакции и внешние ключи. Блокировки на уровне таблиц. Обеспечивает высокую скорость чтения и малое потребление ресурсов. | Хранилища данных (DWH), аналитика, чтение больших объемов данных, системы с редкими операциями записи. |
| **Aria** | Улучшенная версия MyISAM, разработанная для MariaDB. Поддерживает транзакции и автоматическое восстановление после сбоев (crash recovery). Является улучшенной альтернативой MyISAM. | Замена MyISAM в сценариях, где требуется транзакционная безопасность при сохранении высокой скорости чтения. |

**Основные критерии выбора движка:**
- **InnoDB** — выбор по умолчанию для большинства приложений с транзакциями и внешними ключами.
- **MyISAM** — для систем с преобладанием чтения и редкими изменениями (без требований к транзакциям).
- **Aria** — улучшенная альтернатива MyISAM с поддержкой транзакций и восстановлением после сбоев.

**Итог:** Сформировано понимание архитектуры MariaDB, изучены особенности трех основных движков хранения: InnoDB, MyISAM, Aria. Освоен выбор оптимального движка в зависимости от требований к транзакционности, скорости и надежности хранения данных.
